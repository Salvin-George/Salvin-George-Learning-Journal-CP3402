# Content Insights

Through this week’s learning, I developed a clearer understanding of how static websites are structured and why separating responsibilities between technologies is important. HTML is used to define the structure and meaning of content, while CSS controls visual presentation such as layout, colour, and typography. Keeping these concerns separate made it easier to update the design of the site without breaking its structure.

I also learned how Bootstrap’s grid system simplifies responsive layout design by using rows and columns rather than manual positioning. This approach made it easier to create consistent layouts across multiple pages and ensured the site adapts better to different screen sizes. Additionally, deploying the site using GitHub Pages helped me understand how static hosting works and highlighted the importance of correct file organisation and relative paths when publishing a site online.

# Career/Employability/Learning Insights

This week reinforced the importance of following industry best practices even for small projects. Using external CSS files, a consistent navigation structure, and a logical folder hierarchy reflects how real-world web projects are organised and maintained. These practices improve readability, maintainability, and make it easier for other developers to understand and contribute to a project.

From a learning perspective, I found that combining short periods of reading documentation with immediate hands-on practice was far more effective than passively consuming learning material. Building the website first and then consulting documentation to solve specific problems helped me stay engaged and better retain what I learned. This approach feels more aligned with how learning happens in a professional environment and is a strategy I plan to continue using throughout the subject.
